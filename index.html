Reliance Trends Store Sales Forecasting (PAN-India)
Built a 7-day rolling daily sales forecasting system for 1,200 Reliance Trends stores within a broader 3,500-store retail network. Data spanned from Sep 2021 with ~4.5M total rows across monthly Excel files, supplemented by a store master and promotional offers calendar. Business teams used forecasts to identify expected shortfalls and trigger targeted promotional campaigns before month-end to meet sales targets. Baseline univariate models (ARIMA, SARIMA, FBProphet, LSTM) gave 45–55% MAPE accuracy. Shifted to multivariate models incorporating LWSales, LMSales, weekday, and festive flags — running separate models for PAN-India and each state. Identified regional festive effects per state; engineered positive pre-festive and negative post-festive weightages to capture human buying behavior accurately. After 1 year of live forward testing, finalized SARIMAX and LSTM as production models, achieving 90–95% accuracy nationally and 85–95% at state level (peak: 97–98%). Deployed via a Pydash UI connected to the data engineering pipeline on Azure Cloud.

81% Directional Accuracy · 52 Macro Features · EIA Dataset
Brent Oil Spot Price Prediction
Short-term (monthly) and long-term (quarterly) crude oil price forecasting for the petrochemical business unit to support procurement strategy and market planning. Dataset: 52 macro-economic variables (supply, demand, inventory, production, consumption) from the EIA — Jan 1997 to Jul 2024 (330 months / 110 quarters). Standard feature selection (forward selection, backward elimination, VIF) failed due to high multicollinearity in supply-demand variables. Solved using Lasso regression to auto-select important features, followed by domain-driven grouping (supply / demand / inventory / consumption) and within-group VIF pruning. Final features: OPEC Production, Non-OPEC Production, OECD/Non-OECD Consumption, Crude Oil Inventory, Liquid Fuel Supply, Heating/Cooling Degree Days. Achieved 81% directional price-movement accuracy using SVM and Logistic Regression classifiers.

30,000+ Reviews/Month · On-Premise LLM · Data-Secure · Azure
Zero-Shot Classification & Review Summarization
Petrochemical business teams generated 30,000+ site-visit reviews per month exported as CSV/Excel. Senior management needed automated classification, summarization, and a queryable interface — without sending confidential data to external LLM APIs. Built an end-to-end Gradio application powered by Llama 3.2 (9B) via Ollama, deployed on an Azure Linux server — one model for the entire application, with prompts and output formatting changed per task. Features: file upload (CSV/Excel/text) → zero-shot classification table (price / demand / supply / inventory / positive-negative per review) → dual summarization (direct summary + map-reduce for large review volumes) → RAG-style chatbot querying the full review corpus. Earlier transformer/BERT approaches underperformed on large text volumes; Llama 3.2 delivered superior results. Client demo received strong approval; data security concerns fully resolved by keeping data on the internal server.
